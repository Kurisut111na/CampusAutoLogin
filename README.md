# 校园网自动登录

Dr.COM 校园网认证系统的 Windows 桌面自动登录工具。支持自动登录、心跳监测、断线重连、系统托盘驻留、开机自启、启动最小化。

## 快速开始

1. 双击 `CampusAutoLogin.exe`
2. 输入学号、密码，选择运营商（移动/电信/联通）
3. 点击「登录」
4. 勾选「启动时自动登录」+「记住密码」——下次启动自动登录

关闭窗口会最小化到系统托盘（**不会退出**）。右键托盘图标可显示窗口 / 重连 / 打开日志 / 退出。

## 功能

| 功能 | 说明 |
|------|------|
| **自动登录** | 启动后等待 3 秒（让网卡初始化），然后自动探测网关并发起登录 |
| **心跳检测** | 周期性 HTTP HEAD 请求，检测公网连通性，可自定义检测网址 |
| **断线重连** | 连续 2 次心跳失败判定断线，进入指数退避重连序列（1s/2s/4s/8s/16s，最多 5 次） |
| **DNS 预热** | 登录成功后自动执行 4 轮 DNS/ARP 刷新 + 3 次 HTTP 请求（百度→必应→网关），加速网络就绪 |
| **系统托盘** | 关闭窗口最小化到托盘，三色图标（绿=已连接 / 红=已断开 / 蓝=已登录） |
| **开机自启** | 通过注册表 `Run` 键实现（高级设置中启用） |
| **启动最小化** | 开机自动登录时不弹出窗口，静默驻留托盘（高级设置中启用） |
| **手动重连** | 点击重连按钮立即触发登录（2 秒冷却，防重复点击） |
| **网关自适应** | 有线（10.0.1.5）/ 无线（1.2.3.4）自动探测，支持手动指定 |
| **密码记忆** | 再次启动时自动填入已保存的账号密码 |

## 高级设置

| 设置项 | 说明 |
|--------|------|
| 自定义网关 | 手动指定 Dr.COM 网关 IP，留空则自动探测 |
| 心跳检测开关 | 是否启用定期网络连通性检测 |
| 检测间隔 | 15 ~ 300 秒，默认 45 秒 |
| 检测网址 | 每行一个 URL，心跳检测轮流访问，默认百度+必应 |
| 开机自启动 | 写入 Windows 注册表 `Run` 键，开机自动启动程序 |
| **启动后最小化到托盘** | 开机自动登录时静默运行，不弹出窗口 |

## 自动登录流程

```
应用启动
  │
  ├─ 等待 3 秒（让网卡完成初始化）
  │
  ├─ 探测网关（有线 10.0.1.5 / 无线 1.2.3.4）
  │     └─ 未就绪 → 重试（最多 15 次，每次 3 秒 = 最长 45 秒）
  │
  ├─ 发送 Dr.COM 登录请求（HTTP GET + 账号密码）
  │
  ├─ 登录成功 → DNS 预热序列
  │     Round 1 (0s):  DNS 刷新 + ARP 刷新 + HTTP GET baidu.com
  │     Round 2 (3s):  DNS 刷新 + ARP 刷新 + HTTP GET bing.com
  │     Round 3 (5s):  DNS 刷新 + ARP 刷新 + HTTP GET 网关
  │     Round 4 (6s):  DNS 刷新 + ARP 刷新
  │
  └─ 启动心跳检测 → 持续监控网络连通性
```

## 密码安全

### 加密方案：Windows DPAPI

密码使用 Windows **DPAPI (Data Protection API)** 加密后存储，加密密钥由你的 Windows 登录凭据推导，**绑定当前用户和当前机器**。

```
用户密码
   ↓
DPAPI 加密 (CryptProtectData)
   ↓
Base64 编码
   ↓
config.json（密文存储）
```

- **没有外部密钥文件**。无需管理 `aes_key.dat` 之类的东西。
- 加密和解密完全由 Windows 内核处理，密钥从不出现在磁盘上。
- 密码明文只在登录发起 HTTP 请求时短暂存在于内存中（几秒），不持久化。

### 安全性分析

| 攻击场景 | 安全性 |
|----------|--------|
| 拷贝 config.json 到另一台电脑 | **无法解密** — DPAPI 绑定了原机器 |
| 同一台电脑、不同 Windows 用户 | **无法解密** — DPAPI 绑定了原用户 |
| 同一用户、同一台电脑 | 可解密（这正是我们需要的） |
| 有管理员权限 | 理论上可通过进程注入获取内存中的明文，这是所有客户端加密的固有限制 |
| 配置文件被读取 | 只能看到 base64 密文，无法还原密码 |

> **总结**：你的密码在本机受到 DPAPI 保护，不以明文形式存在于磁盘上。唯一的安全前提是保护好你的 Windows 登录密码。

## 常见问题

**Q: 提示「网关尚未就绪」？**
A: 等待几秒让程序探测网关，或在高级设置中手动设置网关 IP。

**Q: 登录失败？**
A: 检查学号、密码、运营商是否正确。查看日志面板获取详细错误（切换过滤器到 ERROR 级别）。

**Q: 登录后过几十秒才能上网？**
A: 正常。程序已在登录后自动执行 DNS 刷新和网络预热（约 20 秒），期间大部分网站已经可以访问。如果个别网站仍然慢，可能是校园网路由收敛较慢，与程序无关。

**Q: 如何彻底退出？**
A: 右键系统托盘图标 → 退出。直接关窗口只是最小化到托盘。

**Q: 如何完全清除数据？**
A: 删除 `%LOCALAPPDATA%\CampusAutoLogin\` 整个目录。如果启用了开机自启，先去高级设置取消勾选。

**Q: 心跳检测会影响网速吗？**
A: 不会。心跳检测使用 HTTP HEAD 请求（只获取响应头，不下载内容），每次流量约 1KB，CPU 占用 < 0.1%。

**Q: 想重装 / 换账号？**
A: 直接删除 `%LOCALAPPDATA%\CampusAutoLogin\config.json`，重新打开程序即可。
